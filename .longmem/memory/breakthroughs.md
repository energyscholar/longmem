# Breakthrough Insights

> **Optional.** This file captures reasoning chains that shifted understanding — not facts or decisions, but moments where the model changed. It fills naturally over sessions.

**Cap:** 15 active insights + 5 killed insights. When full, drop the least-relevant entry.

**Killed insights** (`[KILLED]`) are preserved to prevent re-derivation of dead ends.

---

## Format

```markdown
### BT-NNN: Title (Session N)
[1-2 paragraphs: what the insight is, how it was reached, why it matters]
**Implication:** [What this changes about understanding]
**Connects to:** [Other breakthroughs, corrections, decisions]
```

For killed insights:
```markdown
### BT-KNNN: Title (Session N) [KILLED]
[What was believed and why it was wrong]
**Kill reason:** [What disproved it]
**Lesson:** [What to avoid next time]
```
